# 贡献指南

## 如何添加新软件技能包

### 1. 创建软件包文件夹

在仓库根目录下创建 `<software>-dft-skills/`，内部结构如下：

```
<software>-dft-skills/
├── main/SKILL.md           # 分发器
├── skills/                 # 子技能
├── shared/                 # 模板、脚本、参考资料
└── examples/               # 示例
```

### 2. 编写 main/SKILL.md

分发器文件只负责：
- 列出可用子技能
- 将任务路由到对应子技能
- **不**包含模板细节或输入语法

### 3. 添加子技能

在 `skills/` 下创建扁平的子技能文件夹，每个子技能的 `SKILL.md` 实现单一功能。

### 4. 填充 shared/

将模板、脚本、参考资料放入 `shared/` 文件夹，确保技能包可以独立使用（不依赖全局 `shared-core/`）。

### 5. 添加示例

在 `examples/` 下放置完整的示例输入/输出文件。

## 开发规范

- **无深层嵌套** — 保持目录扁平
- **无交叉依赖** — 不同软件包之间零依赖
- **无绝对路径** — 全部使用相对路径
- **功能不重复** — 子技能之间不能有重复功能
- **遵循最佳实践** — ORCA 5+、VASP 6+、QE 最新版

## 提交 Pull Request

1. 创建新分支
2. 在分支上完成开发
3. 提交时写清楚变更目的和范围
4. 创建 Pull Request
