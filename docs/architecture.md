# 架构说明

## 核心设计原则

### 1. 单仓库多软件

所有 DFT 软件技能包（ORCA / VASP / QE / Gaussian / CP2K）作为顶级文件夹存放在同一个仓库中。

### 2. 完全独立可移植

每个软件文件夹都是 100% 自包含的，可以独立拷贝使用，不依赖仓库的其他部分。

### 3. 内部 shared/ 文件夹

每个软件包有自己的 `shared/` 文件夹，存放模板、脚本和参考资料。仅使用相对路径。

### 4. 无跨软件依赖

软件包之间没有任何交叉依赖。用户只需安装自己需要的软件技能。

### 5. 单一职责

每个子技能只做一件事：输入生成 / 错误修复 / 输出解析，不混用功能。

## 固定目录结构

所有软件包遵循相同的内部结构：

```
xxx-dft-skills/
├── main/SKILL.md           # 分发器：列出子技能、分配任务，不写模板细节
├── skills/                 # 子技能（扁平，无嵌套）
│   └── <sub-skill>/SKILL.md   # 每个子技能只实现单一功能
├── shared/                 # 模板、脚本、参考资料（打包进技能包保证可移植性）
│   ├── templates/          # .inp 输入模板
│   ├── references/         # 快速参考文档
│   └── scripts/            # 辅助脚本
└── examples/               # 示例输入/输出
```

## SKILL.md 规则

### main/SKILL.md（分发器）

- 仅分发任务、整合输出、列出可用子技能
- **不**编写任何模板细节或输入语法

### Sub-skill SKILL.md（子技能）

- 只实现单一功能
- 遵循各 DFT 软件的官方最佳实践（ORCA 5+、VASP 6+、QE 最新）
- 输出结果可直接用于学术研究和论文

## 禁止行为

- 深层嵌套文件夹
- 跨软件依赖
- 绝对路径
- 子技能之间的功能重复
- 任意自定义文件/文件夹
- 修改固定结构
